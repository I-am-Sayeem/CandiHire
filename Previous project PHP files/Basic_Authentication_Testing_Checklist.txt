================================================================================
                        CANDIDATE AUTHENTICATION TESTING
================================================================================

CANDIDATE REGISTRATION TESTING:
□ Test registration with all valid required fields:
  - Full Name: "John Doe"
  - Email: "john.doe@example.com"
  - Phone Number: "1234567890"
  - Work Type: "full-time" (test all options: part-time, contract, freelance, internship, fresher)
  - Password: "password123" (8+ characters)
  - Confirm Password: "password123"
  - Skills: "PHP, JavaScript, MySQL"

□ Test registration with missing required fields:
  - Missing Full Name
  - Missing Email
  - Missing Phone Number
  - Missing Work Type
  - Missing Password
  - Missing Confirm Password

□ Test email validation:
  - Invalid email format: "invalid-email"
  - Invalid email format: "test@"
  - Invalid email format: "@example.com"
  - Valid email: "test@example.com"

□ Test password validation:
  - Password too short: "1234567" (less than 8 characters)
  - Password mismatch: password="test123", confirm="test456"
  - Valid password: "password123"

□ Test work type validation:
  - Valid: "full-time", "part-time", "contract", "freelance", "internship", "fresher"
  - Invalid: "invalid-type"

□ Test duplicate email prevention:
  - Try registering with existing candidate email
  - Try registering with existing company email
  - Verify appropriate error messages

□ Test successful registration:
  - Verify success message: "Account created successfully!"
  - Verify candidate ID is returned
  - Check database entry is created correctly

CANDIDATE LOGIN TESTING:
□ Test login with valid credentials:
  - Use registered email and password
  - Verify success message: "Welcome back, [Name]! Login successful."
  - Verify candidate ID and full name returned
  - Check session is set correctly

□ Test login with invalid credentials:
  - Wrong email, correct password
  - Correct email, wrong password
  - Non-existent email
  - Empty email field
  - Empty password field

□ Test email format validation:
  - Invalid email format in login
  - Valid email format

□ Test session management:
  - Verify user stays logged in after page refresh
  - Test logout functionality
  - Verify session data is properly stored

================================================================================
                        SECURITY TESTING
================================================================================

BASIC SECURITY CHECKS:
□ Test SQL injection prevention:
  - Try SQL injection in email field: "'; DROP TABLE users; --"
  - Try SQL injection in password field
  - Verify queries are properly parameterized

□ Test XSS prevention:
  - Try XSS in name fields: "<script>alert('XSS')</script>"
  - Try XSS in email field
  - Verify input is properly sanitized

□ Test password security:
  - Verify passwords are hashed in database
  - Test password verification works correctly
  - Check password hashing algorithm (should be PASSWORD_DEFAULT)

□ Test session security:
  - Verify session data is properly validated
  - Test session timeout (if implemented)
  - Check session hijacking prevention

□ Test input validation:
  - Test special characters in all fields
  - Test very long input strings
  - Test empty strings and whitespace

□ Test error message clarity:

  - Messages should not expose system information

□ Test error logging:
