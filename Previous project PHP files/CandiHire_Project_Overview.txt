1. PROJECT OVERVIEW
===================

CandiHire is a comprehensive web-based recruitment management system designed to 
streamline the hiring process for both job seekers and employers. The platform 
integrates modern web technologies with AI-powered features to create an efficient, 
user-friendly environment for job matching, application management, and candidate 
evaluation.

The system serves as a centralized platform where:
- Candidates can create profiles, apply for jobs, take assessments, and track applications
- Companies can post jobs, review applications, conduct interviews, and manage recruitment
- AI algorithms provide intelligent candidate matching and CV analysis
- Comprehensive assessment tools enable skill evaluation and testing

Key Features:
- Dual user interface (Candidate & Company dashboards)
- AI-powered candidate matching system
- Comprehensive exam and assessment platform
- Interview scheduling and management
- Real-time notifications and status tracking
- CV builder and analysis tools
- Secure authentication and session management

2. SYSTEM OBJECTIVES
====================

Primary Objectives:
- Streamline the recruitment process for both candidates and companies

Secondary Objectives:

8. SAMPLE SQL QUERIES
=====================

8.1 User Authentication Query
-----------------------------
```sql
SELECT CandidateID, FullName, Email, Password, IsActive 
FROM candidate_login_info 
WHERE Email = ? AND IsActive = 1;
```

8.2 Job Search Query
--------------------
```sql
SELECT jp.JobID, jp.JobTitle, jp.Location, jp.JobType, 
       jp.SalaryMin, jp.SalaryMax, c.CompanyName, c.Logo
FROM job_postings jp
JOIN Company_login_info c ON jp.CompanyID = c.CompanyID
WHERE jp.Status = 'active' 
  AND (jp.JobTitle LIKE ? OR jp.Skills LIKE ?)
  AND jp.Location LIKE ?
ORDER BY jp.PostedDate DESC;
```

8.3 Application Status Query
----------------------------
```sql
SELECT ja.ApplicationID, ja.Status, ja.ApplicationDate,
       jp.JobTitle, c.CompanyName, c.Logo
FROM job_applications ja
JOIN job_postings jp ON ja.JobID = jp.JobID
JOIN Company_login_info c ON jp.CompanyID = c.CompanyID
WHERE ja.CandidateID = ?
ORDER BY ja.ApplicationDate DESC;
```

8.4 AI Matching Query
---------------------
```sql
SELECT c.CandidateID, c.FullName, c.Skills, c.YearsOfExperience,
       (CASE 
         WHEN FIND_IN_SET('PHP', c.Skills) > 0 THEN 20 ELSE 0 END +
        CASE 
         WHEN FIND_IN_SET('MySQL', c.Skills) > 0 THEN 20 ELSE 0 END +
        CASE 
         WHEN c.YearsOfExperience >= 3 THEN 30 ELSE 0 END +
        CASE 
         WHEN c.Location LIKE '%Remote%' THEN 30 ELSE 0 END
       ) AS MatchScore
FROM candidate_login_info c
WHERE c.IsActive = 1
  AND (FIND_IN_SET('PHP', c.Skills) > 0 OR FIND_IN_SET('MySQL', c.Skills) > 0)
ORDER BY MatchScore DESC;
```

8.5 Exam Results Query
----------------------
```sql
SELECT ea.AttemptID, ea.Score, ea.TotalQuestions, ea.CorrectAnswers,
       e.ExamTitle, es.ScheduledDate, es.ScheduledTime
FROM exam_attempts ea
JOIN exams e ON ea.ExamID = e.ExamID
JOIN exam_schedules es ON ea.ScheduleID = es.ScheduleID
WHERE ea.CandidateID = ? AND ea.Status = 'completed'
ORDER BY ea.StartTime DESC;
```

8.6 Interview Schedule Query
---------------------------
```sql
SELECT i.InterviewID, i.InterviewTitle, i.InterviewType, i.InterviewMode,
       i.ScheduledDate, i.ScheduledTime, i.Location, i.Platform,
       jp.JobTitle, c.CompanyName
FROM interviews i
LEFT JOIN job_postings jp ON i.JobID = jp.JobID
JOIN Company_login_info c ON i.CompanyID = c.CompanyID
WHERE i.CandidateID = ? AND i.Status = 'scheduled'
ORDER BY i.ScheduledDate, i.ScheduledTime;
```

8.7 Dashboard Statistics Query
-----------------------------
```sql
SELECT 
    (SELECT COUNT(*) FROM job_applications WHERE CandidateID = ?) as TotalApplications,
    (SELECT COUNT(*) FROM interviews WHERE CandidateID = ?) as TotalInterviews,
    (SELECT COUNT(*) FROM exam_attempts WHERE CandidateID = ?) as TotalExams,
    (SELECT COUNT(*) FROM job_applications WHERE CandidateID = ? AND Status = 'interview-scheduled') as PendingInterviews;
```
